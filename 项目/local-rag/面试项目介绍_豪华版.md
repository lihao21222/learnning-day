## 🎓 面试必备知识点

### RAG 技术原理

**什么是 RAG？**
检索增强生成（Retrieval-Augmented Generation）
- 通过检索找到相关文档
- 将文档作为上下文传给 LLM
- 基于检索内容生成回答

**为什么用 RAG？**
1. 知识更新快，不需要重新训练
2. 可以使用企业内部数据
3. 答案可溯源，减少幻觉
4. 数据安全，不需要上传

## ❓ 面试常见问题与回答

### Q1: 为什么选择 Streamlit 做界面？

**回答思路**:
- Streamlit 专门为数据/AI 应用设计
- 快速原型开发，代码量少
- 原生支持聊天界面组件
- 面试演示效果好，有视觉冲击力

**示例回答**:
"Streamlit 是一个非常适合 AI 应用快速开发的框架。选择它主要有几个原因：
1. **开发效率高** - 几百行代码就能做一个完整的 Web 应用
2. **原生支持聊天** - st.chat_message、st.chat_input 这些组件专门为对话场景设计
3. **演示效果好** - 面试时能直接展示可视化界面，比纯命令行更有说服力
4. **数据科学生态好** - 和 pandas、matplotlib 这些工具无缝集成

当然，生产环境可能会用 FastAPI + React，但面试展示 Streamlit 是最佳选择！"

---

### Q2: 如何支持多种文档格式？

**回答思路**:
- 设计模式（策略模式）
- 抽象基类
- 插件化扩展
- 实际代码示例

**示例回答**:
"我用了策略模式来设计文档加载器。首先定义一个抽象基类 `BaseDocumentParser`，然后针对不同格式写具体的解析器：

```python
class BaseDocumentParser(ABC):
    @abstractmethod
    def parse(self, file_path):
        pass

class PDFParser(BaseDocumentParser):
    def parse(self, file_path):
        # PDF 解析逻辑
```

然后用一个 `DocumentLoaderPro` 来统一管理，根据文件扩展名自动选择对应的解析器。这样设计的好处是，以后要支持 EPUB 或者 MOBI，只需要加一个新的解析器类，不需要改现有代码，符合开闭原则！"

### Q3: 项目有哪些可以改进的地方？

**回答思路**:
- 展示深入思考
- 分阶段、有条理
- 结合当前技术趋势

**示例回答**:
"这个项目还有很多可以改进的空间，我是按阶段规划的：

**短期优化**（1-2周）:

2. 支持多模态 - 除了文本，还能处理图片、表格
3. 对话记忆管理 - 更智能的上下文处理

**长期规划**（3-6月）:
1. 分布式部署 - 支持大规模知识库
2. 用户管理 - 多租户、权限控制
3. 知识库更新 pipeline - 自动同步、增量索引

其实我已经有一些实现思路了，比如向量检索部分，可以用 sentence-transformers 生成 embedding，然后用 Chroma 存下来..."

### Q4: 你在做这个项目时遇到的最大挑战是什么？

**回答思路**:
- 真实的困难
- 如何解决的
- 学到了什么

**示例回答**:
"我遇到的最大挑战是**如何让项目既有展示性又不过度复杂**。

一开始我想把所有功能都加上——向量数据库、LLM 集成、完整的生产级代码，但发现这样一来项目太复杂，面试时很难讲清楚，而且依赖太多容易出问题。

后来我采用了**分层设计**的思路：
1. 先做一个纯 Python 的简化版，不依赖外部库，保证能跑通
2. 然后在这个基础上，做架构优化，模块化
3. 最后加上 Web 界面作为展示层

而且我特意保留了多个版本：
- `simple_rag.py` - 最简单，展示核心逻辑
- `advanced_rag.py` - 架构完整，展示工程能力
- `app_simple.py` - Web 界面，展示产品能力

这样面试时可以根据面试官的兴趣，选择合适的版本深入讲！"
